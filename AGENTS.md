# Инструкция для Codex

Ты выполняешь рейтинговую работу по дисциплине «Проектная деятельность в ИТ».

## Материалы

Сначала изучи все документы:
- docs/Оформление рейтинговой работы (9).docx
- docs/РР1_Проектная деятельность в ИТ (2).docx
- example/Грузин МД — пример прошлой рейтинговой работы

Пример использовать только как образец структуры и уровня оформления. Содержание новой работы должно соответствовать заданию по Git/GitHub.

## Главная цель

Подготовить полную рейтинговую работу:
1. создать ИТ-проект по варианту;
2. создать структуру проекта и исходный код;
3. настроить локальный Git-репозиторий;
4. создать ветки по ИСР и модели жизненного цикла ИС;
5. выполнить Git-команды, требуемые в задании;
6. подключить публичный GitHub-репозиторий;
7. подготовить скриншоты;
8. собрать итоговый DOCX-отчёт.

## Обязательные Git-команды, которые надо показать в работе

- git config
- git init
- git status
- git add
- git commit
- git branch
- git checkout или git switch
- git stash
- git log
- git remote
- git push
- git clone
- git fetch
- git pull

## Важно

Не создавай фиктивные скриншоты.
Если нужен скриншот, остановись и скажи мне, какой экран нужно снять.

Не публикуй ничего в GitHub без моего подтверждения.

Не меняй личные данные в титульном листе на вымышленные. Если данных не хватает, задай вопросы.

## Требования к отчёту

Итоговый файл должен быть DOCX.
Название файла: Фамилия_И.О_git.docx

Оформление:
- Times New Roman;
- размер 14;
- интервал 1,5;
- поля: левое 3 см, правое 1,5 см, верхнее 2 см, нижнее 2 см;
- титульный лист;
- оглавление;
- введение;
- основания для разработки и назначение разработки;
- основная часть по созданию и настройке Git/GitHub;
- ИСР по проекту и ветки по модели жизненного цикла ИС;
- настройка и подключение локального репозитория;
- заключение;
- список источников;
- приложения со скриншотами при необходимости.

## Сначала сделай только анализ

Перед выполнением:
1. перескажи задание;
2. перечисли недостающие данные;
3. предложи вариант ИТ-проекта;
4. предложи структуру папок;
5. предложи структуру веток Git;
6. предложи план коммитов;
7. дождись моего подтверждения.
